# 编码规范

- 使用 Python 3，缩进为 4 个空格，遵循 PEP 8 的基本布局。
- GPU环境运行在系统的: `conda activate tread`
- 导入顺序：标准库、第三方库、本项目模块。
- 命名：类用 `PascalCase`；函数、变量用 `snake_case`；常量用 `UPPER_CASE`。
- 不添加无必要的 CLI 参数；默认实验参数要写在脚本常量或配置文件中,从而使得代码可以直接运行。
- 鼓励代码简洁实现：优先使用直接可读的函数；不要为了兼容而引入复杂抽象或多层间接调用。
- 不保留废弃 wrapper、旧兼容分支、无意义 fallback、未使用字段、未使用配置、未使用函数和未使用脚本；发现后应直接删除，而不是继续转发或隐藏在兼容层后。
- 注释只解释不明显的设计或约束，避免重复代码含义。避免无意义的解释,不要造词,而是使用易懂简洁的用词
- 不使用空 `except`；错误应直接暴露或带有明确上下文。
- 不提交 `__pycache__`、大型可再生成结果和临时文件。
- 在不破坏现有功能和配置字的前提下，删除冗余代码、重复逻辑、无用接口和临时缓存。
- 复用的工具函数统一放在根目录`tools/`
